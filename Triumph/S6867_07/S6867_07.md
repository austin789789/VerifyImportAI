# 1 Overview

The Trip Computer Distance to Empty function calculates an estimate of the number of miles or kilometres, which can be travelled on the remaining volume of fuel in the tank.

After battery connection “Distance to Empty” uses a starting value for the “Instantaneous Fuel Consumption” (determined by diagnostic command, default 4.70l/100km, equivalent to 50 mpg uk/60 mpg us).

The calculation will become more accurate over time, such that “Distance to Empty” will be calculated based on the riding style and economy of the individual rider.

# 8 Error Handling and Recovery Conditions

The following Error Handling shall be carried out:

1. $\varnothing I F C _{n _ { C a l c} }$ shall be frozen if:

• CAN message containing the signal “Vehicle Speed” is missing after a valid value has been received, or the signal is out of range,   
• CAN message containing the signal “Fuel consumption” is missing after a valid value has been received, or the signal is out of range.

2. The display shall show no distance value (instead dashes, i.e. “- - -“, shall be displayed) if:

• CAN message containing the signal “Vehicle Speed” is missing at ignition on,   
• CAN message containing the signal “Fuel consumption” is missing at ignition on,   
• CAN message containing signal “Fuel remaining (litres)” is missing or the signal is out of range,   
• There is an error calculating the value of distance to empty,   
$\bullet \varnothing I F C _{n _ { C a l c} }$ is calculated to be 0 (zero),   
• There is an EEPROM error relating to the distance to empty function.
